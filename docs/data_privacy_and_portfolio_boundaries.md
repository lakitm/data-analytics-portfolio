# Data Privacy & Portfolio Boundaries

This portfolio is designed to show capability without exposing protected work.

## Publicly Safe to Include

- project summaries;
- sanitized screenshots;
- synthetic data schemas;
- aggregate evaluation metrics;
- high-level methodology;
- pseudocode;
- public-dataset notebooks;
- explanations of personal contribution.

## Do Not Include Publicly

- raw health records;
- row-level disease surveillance data;
- private institutional datasets;
- company financial records;
- full private thesis notebooks;
- service keys, API keys, database URLs, or credentials;
- unpublished full thesis manuscript without permission;
- code or assets owned jointly by a team, school, employer, client, or public office.

## Recommended Protection Approach

Use the public repository as a recruiter-facing case-study portfolio, not as a full source-code archive.

For sensitive projects:
- show the problem, method, tools, model comparison, and outcome;
- exclude raw data and complete implementation;
- use synthetic sample schemas;
- add a clear repository license;
- keep deployment credentials and raw files private.
