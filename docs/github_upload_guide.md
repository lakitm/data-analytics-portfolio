# GitHub Upload Guide for Beginners

## Key Terms

- Repository: the project folder stored on GitHub.
- README.md: the front page of the repository.
- Commit: a saved change with a message.
- Branch: a separate version of the repository. Use `main` for your final public version.
- Fork: someone else's copy of your repository under their own GitHub account.
- Clone: a local copy of a repository on your computer.
- Pull request: a request to merge changes from one branch or fork into another.
- Star: a bookmark or public endorsement.
- Watch: notification subscription.
- Pin: a repository featured on your GitHub profile.

## Upload Steps

1. Open your `Portfolio` repository.
2. Click `Add file`.
3. Choose `Upload files`.
4. Drag the files and folders from this pack.
5. Write a clear commit message, such as:
   `Build recruiter-facing data analytics portfolio structure`
6. Click `Commit changes`.
7. Open the repository page and check whether the README displays correctly.
8. Go to the repository `About` section and update the description:
   `Data analytics, forecasting, machine learning, and dashboard case studies.`
9. Add topics:
   `data-analysis`, `python`, `machine-learning`, `forecasting`, `dashboard`, `statistics`, `sql`, `portfolio`
10. Pin the repository on your profile.

## Before Uploading

Check that there are no:
- CSV files with private data;
- `.env` files;
- keys or tokens;
- private Colab links;
- unredacted admin screenshots;
- full unpublished thesis PDFs;
- company or institutional records.
